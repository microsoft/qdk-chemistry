# How-To: SOSSA Resource Estimation with QDK Chemistry

**Branch:** `feature/cyr/sossa`
**PR:** [#539 — SOSSA hamiltonian builder and circuit mapper](https://github.com/microsoft/qdk-chemistry/pull/539)

This guide walks through using QDK Chemistry to build Sum-of-Squares Spectral Amplification (SOSSA) block-encoding circuits from factorized Hamiltonians, verify them via QPE simulation, and produce fault-tolerant resource estimates using QRE v3.

## 2. Prepare Input Data

SOSSA operates on a **DFTHC (Density-Fitted Tensor Hypercontraction) factorized Hamiltonian** stored as a `FactorizedHamiltonianContainer`. The container holds:

| Component | Shape | Description |
|-----------|-------|-------------|
| `h1` | $(N, N)$ | One-body integrals |
| `u_matrices` | $(R \cdot B \cdot N,)$ | Orbital rotation vectors |
| `w_matrices` | $(R \cdot B \cdot C,)$ | Two-body coefficients |
| `wb_matrix` | $(R, C)$ | One-body correction coefficients |

where $N$ = number of orbitals, $R$ = ranks, $B$ = bases, $C$ = copies.

### Load from JSON

```python
from qdk_chemistry.data import Hamiltonian

hamiltonian = Hamiltonian.from_json("path/to/molecule.hamiltonian.json")
factorized_hamiltonian = hamiltonian.get_container()
```

A small example file is included at `examples/h2_factorized_r1_b2_c1.hamiltonian.json`.

### Coming from `qdk-chemistry-addons` DFTHC data

If your data was generated by the DFTHC pipeline in `qdk-chemistry-addons`, the mapping from `addons/data/dfthc.hpp` to `FactorizedHamiltonianContainer` is straightforward since both store the same tensor components (`h1`, `u_matrices`, `w_matrices`, `wb_matrix`, etc.). Serialize your addons data into JSON or HDF5 using the Hamiltonian schema.

> Future work: Use `FactorizedHamiltonianContainer` directly in the `qdk-chemistry-addons` DFTHC workflow.

## 3. Build SOSSA Circuit

The SOSSA workflow has two stages:

1. **`SOSSABuilder`** — converts a `FactorizedHamiltonianContainer` into a `UnitaryRepresentation` (containing the SOSSA decomposition: outer/inner PREPARE coefficients, SELECT rotation angles, normalization).
2. **`SOSSAMapper`** — maps the unitary representation into a controlled Q# circuit.

The `SOSSAMapper` has five configurable settings that control the circuit sub-components:

**For simulation** (small systems, exact): use `"dense_pure_state"` + `"direct"` + `"direct"`.
**For resource estimation** (realistic costs): use `"alias_sampling"` + `"controlled_alias_sampling"` + `"qrom_phase_gradient"` with appropriate bit precisions.

| Setting | Type | Default | Options | Description |
|---------|------|---------|---------|-------------|
| `outer_prepare` | `AlgorithmRef` | `("state_prep", "alias_sampling")` | `"alias_sampling"`, `"dense_pure_state"`, `"qrom"` | Amplitude-loading into the outer register. |
| `inner_prepare_algorithm` | `str` | `"controlled_alias_sampling"` | `"controlled_alias_sampling"`, `"direct"` | Inner (conditional) PREPARE over bases. |
| `select_algorithm` | `str` | `"qrom_phase_gradient"` | `"qrom_phase_gradient"`, `"direct"` | SELECT implementation (Givens rotations + Majorana). |
| `rotation_bit_precision` | `int` | `10` | Any positive int | Bits for Givens rotation angles. |
| `coefficient_bit_precision` | `int` | `10` | Any positive int | Bits for alias sampling coefficients. |

### Minimal example

```python
from qdk_chemistry.algorithms.hamiltonian_unitary_builder.block_encoding.sossa import SOSSABuilder
from qdk_chemistry.algorithms.controlled_circuit_mapper.sossa_mapper import SOSSAMapper
from qdk_chemistry.data import AlgorithmRef

# Step 1: Build the SOSSA unitary representation
builder = SOSSABuilder()
unitary_rep = builder.run(factorized_hamiltonian)

# Step 2: Map to a controlled circuit
mapper = SOSSAMapper()

# Configure sub-components (see Settings Reference below)
mapper.settings().set("outer_prepare", AlgorithmRef("state_prep", "dense_pure_state"))
mapper.settings().set("inner_prepare_algorithm", "direct")
mapper.settings().set("select_algorithm", "direct")

circuit = mapper.run(unitary_rep)
```

### Visualize the circuit

```python
from qdk.widgets import Circuit as CircuitWidget
import qsharp
from qdk._interpreter import CircuitGenerationMethod

factory = circuit._qsharp_factory
qs_circuit = qsharp.circuit(
    factory.program,
    *factory.parameter.values(),
    generation_method=CircuitGenerationMethod.Simulate,
)
CircuitWidget(qs_circuit)
```

## 4. Integrate with QPE

The standard QPE circuit builder provides a one-stop interface that chains SOSSABuilder → SOSSAMapper → QPE:

```python
from qdk_chemistry.algorithms.phase_estimation.circuit_builder.standard_builder import QdkStandardQpeCircuitBuilder
from qdk_chemistry.algorithms import create
from qdk_chemistry.data import AlgorithmRef, Configuration, Wavefunction, StateVectorContainer

# Prepare a trial state (e.g. Hartree-Fock)
N = factorized_hamiltonian.get_num_orbitals()
orbitals = factorized_hamiltonian.get_orbitals()
state_prep = create("state_prep", "sparse_isometry_gf2x")
hf_config = Configuration.canonical_hf_configuration(N, N, 2 * N)
hf_wavefunction = Wavefunction(StateVectorContainer(hf_config, orbitals, "electrons"))
state_prep_circuit = state_prep.run(hf_wavefunction)

# Build QPE circuit with SOSSA walk operator
num_phase_bits = 6  # for simulation; increase for precision
qpe_builder = QdkStandardQpeCircuitBuilder(
    num_bits=num_phase_bits,
    controlled_circuit_mapper=AlgorithmRef(
        "controlled_circuit_mapper", "sossa",
        outer_prepare=AlgorithmRef("state_prep", "dense_pure_state"),
        inner_prepare_algorithm="direct",
        select_algorithm="direct",
    ),
    unitary_builder=AlgorithmRef("hamiltonian_unitary_builder", "sossa"),
)

qpe_circuits = qpe_builder.run(
    state_preparation=state_prep_circuit,
    qubit_hamiltonian=factorized_hamiltonian,
)
```

## 5. Resource Estimation with QRE v3

### Choosing parameters for resource estimation

For resource estimation, switch to resource-efficient sub-components and set bit precisions:

| Parameter | Typical Range | Notes |
|-----------|---------------|-------|
| `rotation_bit_precision` (`b_rot`) | 10–20 | Bits for Givens rotation synthesis. Higher = more accurate but more T-gates. |
| `coefficient_bit_precision` (`b_coeff`) | 7–15 | Bits for alias sampling coefficients. |
| `outer_prepare` | `"alias_sampling"` | Resource-efficient state preparation. |
| `inner_prepare_algorithm` | `"controlled_alias_sampling"` | 2D alias sampling. |
| `select_algorithm` | `"qrom_phase_gradient"` | QROM-based SELECT with phase-gradient rotations. |

### Number of queries (phase qubits)

The number of walk-operator queries is determined by the Heisenberg limit:

$$
N_{\text{queries}} = \left\lceil \frac{\pi \lambda_{\text{eff}}}{2 \sigma_E} \right\rceil
$$

where $\lambda_{\text{eff}}$ is the effective normalization and $\sigma_E$ is the target energy precision (e.g., $10^{-3}$ Ha for chemical accuracy). The number of phase qubits is:

$$
n_{\text{phase}} = \left\lceil \log_2 N_{\text{queries}} \right\rceil
$$

> **Known issue:** The relationship between $n_{\text{phase}}$ and $N_{\text{queries}}$ involves subtleties around fractional phase qubits and the exact query-counting convention (powers of 2 vs. direct queries). This is under active investigation. The current implementation uses $2^{n_{\text{phase}}}$ queries.

### Full resource estimation example

```python
import math
from qdk.qre import estimate, plot_estimates
from qdk.qre.models import Majorana, RoundBasedFactory, ThreeAux

# Parameters
lambda_eff = 21.45   # effective normalization (from SOSSA builder)
sigma_E = 1e-3       # target precision in Hartree
b_rot = 15           # rotation bit precision
b_coeff = 15         # coefficient bit precision

# Compute queries
num_queries = math.ceil(math.pi * lambda_eff / (2 * sigma_E))
num_phase_qubits = math.ceil(math.log2(num_queries))

# Build the QPE circuit with resource-efficient settings
qpe_builder = QdkStandardQpeCircuitBuilder(
    num_bits=num_phase_qubits,
    controlled_circuit_mapper=AlgorithmRef(
        "controlled_circuit_mapper", "sossa",
        outer_prepare=AlgorithmRef("state_prep", "alias_sampling"),
        inner_prepare_algorithm="controlled_alias_sampling",
        select_algorithm="qrom_phase_gradient",
        rotation_bit_precision=b_rot,
        coefficient_bit_precision=b_coeff,
    ),
    unitary_builder=AlgorithmRef("hamiltonian_unitary_builder", "sossa"),
)

qpe_circuits = qpe_builder.run(
    state_preparation=state_prep_circuit,
    qubit_hamiltonian=factorized_hamiltonian,
)
qpe_circuit = qpe_circuits[0]

# Get logical resource counts
logical_counts = qpe_circuit.estimate().logical_counts
print(f"Logical Toffoli: {logical_counts['cczCount']}")
print(f"Logical qubits:  {logical_counts['numQubits']}")

# Run QRE v3 physical resource estimation
app = qpe_circuit.get_qre_application()
architecture = Majorana(error_rate=1e-5)
isa_query = ThreeAux.q() * RoundBasedFactory.q(use_cache=True, code_query=ThreeAux.q())
qre_result = estimate(app, architecture, isa_query, max_error=0.01, name="SOSSA QPE")

# Display results
plot_estimates(qre_result, figsize=(6, 4), runtime_unit="ms")
```

## 6. State Preparation Options

| Algorithm | Factory Name | Best For | Key Setting |
|-----------|-------------|----------|-------------|
| Sparse Isometry (GF2+X) | `"sparse_isometry_gf2x"` | Sparse wavefunctions (HF, few-det) | `dense_preparation_method` |
| Alias Sampling | `"alias_sampling"` | Balanced resource cost | `bits_precision` (default 10) |
| Dense Pure State | `"dense_pure_state"` | Small systems / simulation (<32 qubits) | — |
| QROM | `"qrom"` | Circuit depth optimization | `rotation_bit_precision` |

For MPS (Matrix Product State) preparation, see the draft PR for MPS state preparation examples.

## 7. Known Issues and Future Work

1. **Number of queries:** The current implementation uses $2^{n_{\text{phase}}}$ queries. The optimal query count may differ from this power-of-two assumption, and the relationship to fractional phase qubits needs further investigation. This affects comparison with external resource estimates.

2. **Context separation:** The Q# context (`qdk.context`) used for the SOSSA circuit should be separated from the context used for auxiliary circuits (e.g., `qir_to_qiskit` conversions). This is a planned improvement.

3. **DFTHC data integration:** In the future, `addons/data/dfthc.hpp` will be replaced by the `FactorizedHamiltonianContainer` as the canonical data format.
